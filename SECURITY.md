# Security Policy

## Supported Versions

Currently, only the latest major version of `current-week` is supported with security updates. If a security vulnerability is found, a patch will be released for the latest version.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is taken seriously. If you discover a security vulnerability within `current-week`, please do not open a public issue, as that alerts attackers to the vulnerability before a patch can be prepared.

Instead, please report it via one of the following methods:

1. **Email:** Send a detailed report to mattias@makra.org. 
2. **GitHub Private Vulnerability Reporting:** Go to the [Security tab](https://github.com/mazk0/current-week/security/advisories) of this repository, click on "Advisories", and click **"Report a vulnerability"**.

### What to include in your report:
* A description of the vulnerability and its impact.
* Steps to reproduce the vulnerability.
* Any potential mitigation steps you might suggest.

### What to expect:
* You should receive an acknowledgment of your report within **48 hours**.
* We will investigate the issue and determine if it is a valid vulnerability.
* If accepted, we will keep you updated on the progress of the patch and notify you when a fix is released. 
* You will be credited in the release notes and security advisory for the discovery (unless you prefer to remain anonymous).

Thank you for helping keep this project secure!
